# 우리집 재고판

수량을 바로 고치고, 필요할 때 사진과 명칭을 직접 정리하는 가정용 식재료 재고 웹앱입니다.

## 실행

```bash
node server.js 4174
```

브라우저에서 엽니다.

```text
http://localhost:4174
```

같은 와이파이의 다른 기기에서 쓰려면 Mac의 로컬 IP 주소로 접속합니다.

```text
http://<Mac-IP>:4174
```

## 현재 기능

- 현재 Obsidian 재고 기반 초기 데이터
- 재고 전용 화면과 카테고리 탭 기반 컴팩트 카드 보기
- 각 카테고리 안에서 유통기한순 정렬
- 보관 구분은 `냉장`, `냉동`, `실온`만 사용
- 재고 검색 시 `활성만`, `소진 포함`, `소진만` 상태 필터
- 목록에서 `-1`, `+1` 즉시 수량 반영
- `추가하기`에서 이름, 구성/재료, 수량, 보관, 유통기한을 확인한 뒤 최종 추가
- 상세 화면에서 품목명, 사진, 남은 수량 직접 수정
- 말로 사용 내역 입력 후 변경안 반영
- 직접 `-1`, `+1`, `다 씀`, `얼마 안 남음` 처리
- 주문/영수증 텍스트 기반 입고 후보 생성
- 최근 변경 기록
- Markdown 내보내기

## UI 방향

토스 디자인 시스템(TDS) 문서의 모바일 UX 패턴을 참고해 주요 액션은 파란색 fill 버튼, 보조 액션은 weak 버튼, 입력 필드는 조용한 회색 박스, 화면 구조는 모바일 우선 카드/하단 탭으로 정리했습니다. 공식 TDS 패키지를 직접 포함한 것은 아니며, 현재 정적 웹앱 구조에 맞춰 TDS 스타일 원칙을 반영한 MVP입니다.

## 데이터

공유 데이터는 아래 파일에 저장됩니다.

```text
data/state.json
```

현재 버전은 개인/가정용 MVP입니다. 실제 외부 배포에는 로그인, 권한, 이미지 저장소, AI/OCR API 연결이 추가로 필요합니다.

기본 인터넷 썸네일은 사용하지 않습니다. 사진은 추가하기 또는 상세 화면에서 직접 업로드한 것만 표시됩니다.

## Vercel 배포

Vercel에서는 `/api/state` 서버리스 함수가 상태를 저장합니다.

- `BLOB_READ_WRITE_TOKEN`이 있으면 Vercel Blob의 `home-inventory/state.json`에 저장합니다.
- 토큰이 없으면 로컬 개발용 파일 저장으로 동작합니다. Vercel 서버리스 환경에서는 실제 공동 사용을 위해 Vercel Blob 연결이 필요합니다.

배포 전후 Vercel 프로젝트에서 Blob Storage를 연결해 `BLOB_READ_WRITE_TOKEN` 환경 변수가 생기도록 설정하세요.

### GitHub + Vercel 순서

1. GitHub에서 새 저장소를 만듭니다.
2. 이 폴더를 원격 저장소에 push합니다.
3. Vercel에서 `Add New Project`를 누르고 GitHub 저장소를 Import합니다.
4. Vercel 프로젝트의 Storage에서 Blob을 연결합니다.
5. 재배포하면 같은 URL을 여러 기기에서 함께 사용할 수 있습니다.
